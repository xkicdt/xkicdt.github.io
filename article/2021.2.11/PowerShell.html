PowerShell

前言

我写这篇文章是什么目的呢？首先，Windows有90%的市场份额，但是大部分用户都是低端用户，因为微软要抢占市场，所以要把Windows做的简单易用，大部分操作都不需要命令行，只需要在图形界面点几个按钮就行了。但是这样偏偏降低了效率，有时在命令行下只用20个字符就能完成的操作，得在图形界面点30次。这就是为何Windows Server Core砍掉图形界面的原因，因为不需要。还有，图形界面在进行大型的、复杂的操作时易点错，但是确实更加简单，降低运维成本。但这样缺点就显现出来了，Windows Server的运维到大街上随便拉一个就行，所以运维水平普遍低。砍掉了图形界面的Windows Server Core稳定性不比Linux差，至少比Ubuntu Server强，但是低质量运维导致了Windows不稳定的骂名，其实是低质量运维不想承认罢了。Windows也有脚本，比如*.cmd/*.vbs/*.bat，但是很少人用，只有高端用户才用。Windows不比*nix差，什么脚本、命令行操作都一应俱全，并且PowerShell还比多数*nix shell强大，但是会用的人少。说来就是宣传一下PowerShell。

PowerShell语法

PowerShell使用cmdlet形式语法，也就是[动词-名词]式的语法。例如Get-Command，就是获取帮助。同样的，PowerShell也照顾了从cmd或bash转来的用户，例如列出指定目录下的文件在bash里就是ls，在cmd里就是dir，在PowerShell里就是Get-ChildItem，但是你在PowerShell里使用ls或dir也是没有问题的。同时wget、cat、echo等命令也是可以的，它们在PowerShell里分别是Invoke-WebRequest和Get-Content还有Write-Output。

PowerShell常用命令

一个长期使用其他Shell的人可能已经形成了肌肉记忆，清屏要打clear，列出文件要ls。这些在PowerShell里面都可以用。下面是常用命令：

PowerShell bash cmd

Get-ChildItem ls dir

Get-Content cat 未知

Write-Output echo echo

Clear-Host clear cls

Set-Location cd cd

Get-Command 未知 未知

使用Get-Command不加参数，会列出所有可用命令

PowerShell自带了一个[管道]，它比bash的管道强大多了，*nix shell只有Elvish能与它媲美。
